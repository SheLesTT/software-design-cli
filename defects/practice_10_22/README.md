# Диаграмма компонентов

Данная диаграмма отображает архитектуру системы регистрации, учета и устранения дефектов на автомобильном заводе.  
Компоненты системы разделены на **Client UI Core**, **Backend** и **Auxiliary Services**.

---

## Client UI Core
Объединяет интерфейсы взаимодействия с пользователем для четырех ролей:

- **Worker UI** — используется ремонтником для регистрации дефектов и выполнения ремонта.  
- **Dispatcher UI** — обеспечивает диспетчеру контроль свободных ремонтных мест и распределение автомобилей.  
- **Foreman UI** — используется бригадиром для назначения ремонтников.  
- **Quality Service UI** — интерфейс отдела контроля качества.

---

## Backend
Компонент, объединяющий логику всех подсистем, определяющих работу интерфейсов взаимодействия с пользователями.

- **External API Interface** — шлюз, принимающий запросы от UI и перенаправляющий их во внутренние сервисы.  
- **Notification Delivery and Registration Service** — отвечает за уведомления и регистрацию событий.  
- **Repair System** — включает компоненты **Repair Zone** и **Repair Place**, где моделируются ремонтные зоны и рабочие места; объединены в общую подсистему, отвечающую за ремонт.  
- **Reporting Service** — компонент, формирующий отчёты по сменам.  
- **DefectSystem** — регистрирует дефекты и хранит их описание, статус работы над ними.  
- **Control Core** — контролирует взаимодействие всех сервисов и координирует передачу данных между ними.

---

## Auxiliary Services
Компонент системы, объединяющий вспомогательные сервисы для бэкенда.  
Обеспечивает высокую надежность хранения и передачи данных.
